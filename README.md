# Домашнее задание к занятию «Микросервисы: масштабирование» Теплов Михаил

Вы работаете в крупной компании, которая строит систему на основе микросервисной архитектуры.
Вам как DevOps-специалисту необходимо выдвинуть предложение по организации инфраструктуры для разработки и эксплуатации.

## Задача 1: Кластеризация

Предложите решение для обеспечения развёртывания, запуска и управления приложениями.
Решение может состоять из одного или нескольких программных продуктов и должно описывать способы и принципы их взаимодействия.

Решение должно соответствовать следующим требованиям:
- поддержка контейнеров;
- обеспечивать обнаружение сервисов и маршрутизацию запросов;
- обеспечивать возможность горизонтального масштабирования;
- обеспечивать возможность автоматического масштабирования;
- обеспечивать явное разделение ресурсов, доступных извне и внутри системы;
- обеспечивать возможность конфигурировать приложения с помощью переменных среды, в том числе с возможностью безопасного хранения чувствительных данных таких как пароли, ключи доступа, ключи шифрования и т. п.

Обоснуйте свой выбор.

## Вариант 

Для организации развёртывания, запуска и управления микросервисами предлагается использовать Kubernetes как основную платформу оркестрации контейнеров.

---

##  Основной компонент

### Kubernetes

Kubernetes используется как основа инфраструктуры и обеспечивает:
- развёртывание контейнеризированных приложений
- управление жизненным циклом pod’ов
- балансировку нагрузки
- автоматическое восстановление сервисов

---

##  Контейнеризация

### Docker

Используется для:
- упаковки приложений в контейнеры
- обеспечения одинаковой среды dev/stage/prod
- упрощения доставки микросервисов

---

##  Обнаружение сервисов и маршрутизация

Внутренние механизмы Kubernetes:
- CoreDNS — сервис-дискавери
- Service (ClusterIP / NodePort / LoadBalancer) — маршрутизация внутри кластера

Внешний доступ:
- Ingress Controller (NGINX / Traefik) — маршрутизация HTTP/HTTPS запросов

---

##  Масштабирование

### Горизонтальное масштабирование
- Horizontal Pod Autoscaler (HPA)
  - масштабирует количество pod’ов по нагрузке (CPU/RAM/метрики)

### Масштабирование кластера
- Cluster Autoscaler
  - автоматически добавляет или удаляет узлы

---

##  Разделение и безопасность

- Namespaces — логическое разделение окружений (dev / stage / prod)
- Network Policies — контроль сетевого взаимодействия между сервисами
- Ingress + LoadBalancer — управление внешним доступом

---

##  Конфигурации и секреты

- ConfigMap — хранение конфигураций (env-переменные, настройки)
- Secret — хранение чувствительных данных:
  - пароли
  - токены
  - ключи доступа

Дополнительно можно использовать:
- HashiCorp Vault для усиленной безопасности секретов

---

##  Архитектура решения

1. Разработчик собирает контейнер (Docker image)
2. Image отправляется в container registry
3. Kubernetes разворачивает микросервисы
4. Ingress Controller принимает внешний трафик
5. HPA и Cluster Autoscaler обеспечивают масштабирование

---

##  Почему именно такой стек

###  Полное соответствие требованиям
- поддержка контейнеров → Docker
- сервис-дискавери → CoreDNS
- маршрутизация → Services + Ingress
- горизонтальное масштабирование → HPA
- автоматическое масштабирование → Cluster Autoscaler
- изоляция → Namespaces + Network Policies
- конфигурации и секреты → ConfigMap + Secret

---

###  Преимущества 

- индустриальный стандарт cloud-native архитектуры
- работает в облаке и on-premise
- высокая отказоустойчивость
- автоматическое восстановление сервисов
- гибкое масштабирование под нагрузку
